# Архитектурный документ: Калькулятор накоплений на вклады

## 1. Обзор системы

### 1.1 Назначение
Веб-приложение для планирования накоплений с использованием краткосрочных вкладов (3 месяца) с капитализацией процентов и регулярными пополнениями.

### 1.2 Основные возможности
- Расчет итоговой суммы накоплений на заданную дату
- Моделирование циклов 3-месячных вкладов
- Учет изменяющихся процентных ставок по периодам
- Ежемесячное начисление процентов с капитализацией
- Добавление ежемесячных накоплений при открытии нового вклада
- Детальный числовой отчет по результатам

## 2. Функциональные требования

### 2.1 Входные параметры
- **Начальная сумма** (₽): первоначальный капитал для открытия вклада
- **Целевая дата**: дата планируемого изъятия всех средств
- **Ежемесячный взнос** (₽): сумма, откладываемая каждый месяц для добавления к новому вкладу
- **Периоды процентных ставок**: список периодов с указанием:
  - Начальный месяц действия ставки
  - Конечный месяц действия ставки
  - Годовая процентная ставка (%)

### 2.2 Расчетная логика

#### 2.2.1 Цикл вклада (3 месяца)
1. Открытие вклада с суммой = (остаток предыдущего вклада + проценты) + (ежемесячный взнос × 3)
2. Фиксация процентной ставки на момент открытия
3. Ежемесячное начисление процентов: `сумма × (ставка / 12)`
4. Капитализация: проценты добавляются к телу вклада
5. По окончании 3 месяцев цикл повторяется

#### 2.2.2 Особые случаи
- **Неполный последний цикл**: если до целевой даты осталось менее 3 месяцев, вклад открывается на оставшийся срок
- **Изменение ставки**: ставка применяется к тем вкладам, которые открываются в период её действия

### 2.3 Выходные данные (отчет)
- Итоговая сумма на целевую дату
- Общая сумма внесенных средств (начальная + все взносы)
- Общая сумма начисленных процентов
- Эффективная доходность (%)
- Детализация по циклам вкладов:
  - Номер цикла
  - Дата открытия
  - Дата закрытия
  - Сумма при открытии
  - Действующая ставка
  - Начисленные проценты за цикл
  - Сумма при закрытии

## 4. Структуры данных

### 4.1 Входные данные

```typescript
interface InputParameters {
  initialAmount: number;           // Начальная сумма
  targetDate: Date;                // Целевая дата
  monthlyContribution: number;     // Ежемесячный взнос
  ratePeriods: RatePeriod[];       // Периоды процентных ставок
}

interface RatePeriod {
  id: string;                      // Уникальный идентификатор
  startMonth: Date;                // Начало периода (первое число месяца)
  endMonth: Date;                  // Конец периода (последнее число месяца)
  annualRate: number;              // Годовая ставка (например, 16.5 для 16.5%)
}
```

### 4.2 Результаты расчета

```typescript
interface CalculationResult {
  finalAmount: number;             // Итоговая сумма
  totalContributions: number;      // Сумма всех взносов
  totalInterest: number;           // Сумма всех процентов
  effectiveYield: number;          // Эффективная доходность (%)
  cycles: DepositCycle[];          // Детализация по циклам
}

interface DepositCycle {
  cycleNumber: number;             // Номер цикла
  openDate: Date;                  // Дата открытия
  closeDate: Date;                 // Дата закрытия
  openingAmount: number;           // Сумма при открытии
  appliedRate: number;             // Применяемая ставка
  interestEarned: number;          // Начисленные проценты
  closingAmount: number;           // Сумма при закрытии
  monthlyDetails: MonthDetail[];   // Детали по месяцам цикла
}

interface MonthDetail {
  month: Date;                     // Месяц
  balanceStart: number;            // Баланс на начало месяца
  interestAccrued: number;         // Начисленные проценты
  balanceEnd: number;              // Баланс на конец месяца
}
```

## 5. Алгоритм расчета

### 5.1 Псевдокод основного алгоритма

```
ФУНКЦИЯ calculateSavings(parameters):
    currentAmount = parameters.initialAmount
    currentDate = СЕГОДНЯ
    targetDate = parameters.targetDate
    cycles = []
    cycleNumber = 1
    
    ПОКА currentDate < targetDate:
        // Определение длительности цикла
        monthsUntilTarget = разница_в_месяцах(targetDate, currentDate)
        cycleDuration = МИН(3, monthsUntilTarget)
        
        // Добавление накопленных средств
        contributionsToAdd = parameters.monthlyContribution * cycleDuration
        openingAmount = currentAmount + contributionsToAdd
        
        // Определение применяемой ставки
        appliedRate = найти_ставку_на_дату(currentDate, parameters.ratePeriods)
        
        // Расчет по месяцам цикла
        monthlyDetails = []
        balanceStart = openingAmount
        
        ДЛЯ month ОТ 1 ДО cycleDuration:
            monthlyRate = appliedRate / 12 / 100
            interestAccrued = balanceStart * monthlyRate
            balanceEnd = balanceStart + interestAccrued
            
            monthlyDetails.добавить({
                month: currentDate + month,
                balanceStart: balanceStart,
                interestAccrued: interestAccrued,
                balanceEnd: balanceEnd
            })
            
            balanceStart = balanceEnd
        
        // Сохранение результатов цикла
        closingAmount = balanceEnd
        totalInterest = closingAmount - openingAmount
        
        cycles.добавить({
            cycleNumber: cycleNumber,
            openDate: currentDate,
            closeDate: currentDate + cycleDuration месяцев,
            openingAmount: openingAmount,
            appliedRate: appliedRate,
            interestEarned: totalInterest,
            closingAmount: closingAmount,
            monthlyDetails: monthlyDetails
        })
        
        // Переход к следующему циклу
        currentAmount = closingAmount
        currentDate = currentDate + cycleDuration месяцев
        cycleNumber = cycleNumber + 1
    
    // Формирование итогового результата
    finalAmount = currentAmount
    totalContributions = parameters.initialAmount + 
                         (parameters.monthlyContribution * кол-во_месяцев)
    totalInterest = finalAmount - totalContributions
    effectiveYield = (totalInterest / totalContributions) * 100
    
    ВЕРНУТЬ {
        finalAmount,
        totalContributions,
        totalInterest,
        effectiveYield,
        cycles
    }
```

## 6. UI/UX спецификация

### 6.1 Форма ввода

**Секция: Процентные ставки (интерактивный график)**

Визуализация в виде ступенчатого графика:

```

    │

    │                     │

         Окт   Янв   Апр   Июл   Окт
         '25   '26   '26   '26   '26
          ↑     ↑     ↑     ↑     ↑
      (окончания 3-месячных вкладов)
```

**Взаимодействие с графиком:**

1. **Начальное состояние:**
   - График показывает один период с начальной ставкой на весь срок
   - Горизонтальная линия от текущей даты до целевой даты

2. **Добавление точки изменения ставки:**
   - Клик по графику в точке окончания вклада (каждые 3 месяца)
   - График делится на два периода вертикальным разделителем
   - Возможные точки разделения подсвечиваются при наведении

3. **Изменение ставки периода:**
   - Наведение на период → подсветка активного сегмента
   - Перетаскивание периода вверх/вниз → изменение ставки
   - Отображение текущего значения ставки в tooltip при перетаскивании
   - Шаг изменения: 0.25%
   - Диапазон: 1% - 30%

4. **Перемещение точки изменения:**
   - Перетаскивание разделителя влево/вправо
   - Привязка (snap) только к датам окончания вкладов (каждые 3 месяца)

5. **Удаление точки изменения:**
   - Двойной клик на разделителе
   - Или кнопка "×" на разделителе при наведении
   - Периоды объединяются (ставка левого периода распространяется на правый)

**Визуальные элементы:**

- **Сетка:** вертикальные пунктирные линии на датах окончания вкладов
- **Маркеры:** точки на датах окончания вкладов (возможные места разделения)
- **Подсказки при наведении:**
  ```
  ┌─────────────────────┐
  │ Январь 2026         │
  │ Окончание вклада #3 │
  │ Ставка: 18.0%       │
  │ Клик: разделить     │

  ```
- **Подсказки на разделителях:**
  ```
  ┌─────────────────────┐
  │ Апрель 2026         │
  │ Вклад #4            │
  │ 18.0% → 16.0%      │
  │ Тяните: переместить │
  │ ×: удалить         │

  ```

**Дополнительная информация под графиком:**

```
Периоды процентных ставок:
• Вклады #1-3 (Окт 2025 – Июнь 2026): 18.0%
• Вклады #4-6 (Июль 2026 – Март 2027): 16.0%
• Вклады #7-8 (Апр 2027 – Сен 2027): 14.5%

💡 Подсказка: Кликните на дату окончания вклада, чтобы 
   изменить ставку для следующих периодов. Тяните

```

**Валидация:**
- Все поля обязательны
- Начальная сумма и взнос > 0
- Целевая дата > текущей даты
- Ставка в диапазоне 1% - 30%
- Минимальная длина периода: 1 вклад (3 месяца)
- Периоды автоматически покрывают весь срок без пробелов

## 8. Тестовые сценарии

### 8.1 Базовый сценарий
- Начальная сумма: 100 000 ₽
- Целевая дата: через 1 год
- Ежемесячный взнос: 10 000 ₽
- Ставка: 15% годовых (постоянная)

**Ожидаемый результат:**
- 4 полных цикла по 3 месяца
- Итоговая сумма ≈ 248 000 ₽ (примерно)

### 8.2 Сценарий с изменением ставки
- Начальная сумма: 200 000 ₽
- Целевая дата: через 9 месяцев
- Ежемесячный взнос: 20 000 ₽
- Ставки:
  - Месяцы 1-3: 18%
  - Месяцы 4-9: 15%

**Ожидаемый результат:**
- 3 цикла по 3 месяца
- Первый цикл использует ставку 18%
- Второй и третий циклы используют ставку 15%

### 8.3 Граничный случай (неполный цикл)
- Начальная сумма: 50 000 ₽
- Целевая дата: через 5 месяцев
- Ежемесячный взнос: 5 000 ₽
- Ставка: 16% годовых

**Ожидаемый результат:**
- Первый цикл: 3 месяца
- Второй цикл: 2 месяца (неполный)

## 9. Потенциальные расширения

1. **Учет налогообложения** (13%/15% с дохода сверх лимита)
2. **Сравнение стратегий** (3 vs 6 vs 12 месяцев)
3. **Инфляционная корректировка** (реальная vs номинальная доходность)
4. **Оптимизация взносов** (расчет необходимого взноса для достижения целевой суммы)
5. **Интеграция с банковскими API** (автоматическое обновление ставок)
6. **Напоминания** (уведомления о закрытии вклада)
7. **Многовалютность** (расчеты в разных валютах)

## 10. Рекомендации по реализации

1. **Точность вычислений**: использовать библиотеки для работы с decimal числами (например, decimal.js), чтобы избежать ошибок округления
2. **Работа с датами**: всегда нормализовать даты к началу месяца для единообразия
3. **Модульность**: расчетный движок должен быть независим от UI для возможности юнит-тестирования
4. **Производительность**: для длительных периодов (10+ лет) оптимизировать расчеты
5. **UX**: показывать прогресс-индикатор при расчете больших периодов
